# howick-rs

Rust library for parsing and generating **Howick FRAMA machine CSV files** for light gauge steel (LGS) framing.

[![Crates.io](https://img.shields.io/crates/v/howick-rs.svg)](https://crates.io/crates/howick-rs)
[![License](https://img.shields.io/badge/license-MIT%20OR%20Apache--2.0-blue.svg)](LICENSE)

---

## What is a Howick FRAMA Machine?

[Howick Ltd](https://www.howickltd.com) (Auckland, New Zealand) manufactures CNC roll-forming machines that turn coiled steel into precision-cut, pre-punched, pre-labelled C-section framing members for building construction.

The machines consume simple CSV files describing each member:
- Cut length in millimetres
- Positions of dimples, lip cuts, swages, web holes, and end truss cuts

This library lets you **read** those CSV files and **generate** new ones from your own geometry or design data.

---

## Current Software Ecosystem

CSV files for Howick machines are typically generated by compatible framing software including:

- **FrameBuilderMRD** — confirmed in production use (Thailand factory, March 2026)
- Vertex BD
- Tekla Structures (via HowickExport macro)
- AGACAD Metal Framing (Revit plugin)
- Howick's own FRAMA Design Software

This crate provides an **open Rust alternative** — generate Howick CSV directly from your own B-Rep geometry, parametric design tool, or custom pipeline.

## CSV Format

```
UNIT,MILLIMETRE
PROFILE,S8908,Standard Profile
FRAMESET,T1
COMPONENT,T1-1,LABEL_INV,1,3945.0,DIMPLE,20.65,LIP_CUT,23.0,WEB,454.4,...
COMPONENT,T1-2,LABEL_NRM,1,3945.0,DIMPLE,20.65,LIP_CUT,23.0,...
```

Each `COMPONENT` row: `id, label_orientation, quantity, length_mm, [operation, position_mm, ...]`

### Operations

| Operation | Meaning |
|-----------|---------|
| `DIMPLE` | Raised bump for screw location — pairs ~50mm apart at each connection |
| `LIP_CUT` | Cuts the C-section lip at a notch — pairs ~0.16mm apart |
| `SWAGE` | Crimp at chord-to-web joints in trusses |
| `WEB` | Hole through the web for services (pipes, cables, bolts) |
| `END_TRUSS` | Truss end cut angle position |

A real truss CSV (`tests/fixtures/T1_full.csv`) is included as a reference and for tests.

## Usage

```toml
[dependencies]
howick-rs = "0.1"
```

### Parse an existing CSV file

```rust
use howick_rs::csv;

let input = std::fs::read_to_string("T1.csv").unwrap();
let frameset = csv::parse(&input).unwrap();

println!("Frameset: {}", frameset.name);
println!("Profile: {}", frameset.profile.code);
for component in &frameset.components {
    println!("  {} — {}mm, {} operations",
        component.id,
        component.length_mm,
        component.operations.len()
    );
}
```

### Generate a CSV from your own data

```rust
use howick_rs::csv;
use howick_rs::types::{
    Component, Frameset, LabelOrientation, Operation, Profile, Unit,
};

let frameset = Frameset {
    name: "W1".to_string(),
    unit: Unit::Millimetre,
    profile: Profile {
        code: "S8908".to_string(),
        description: "Standard Profile".to_string(),
    },
    components: vec![
        Component {
            id: "W1-1".to_string(),
            label: LabelOrientation::Normal,
            quantity: 1,
            length_mm: 2400.0,
            operations: vec![
                Operation::Dimple(20.0),
                Operation::Dimple(70.0),
                Operation::LipCut(23.0),
                Operation::LipCut(2377.0),
                Operation::Dimple(2330.0),
                Operation::Dimple(2380.0),
            ],
        },
    ],
};

let csv_output = csv::serialize(&frameset).unwrap();
std::fs::write("W1.csv", csv_output).unwrap();
```

## Future: OPC UA Integration

The current Howick machine workflow uses USB sticks or local file shares.
The roadmap includes replacing this with an OPC UA edge agent running on a small compute
module (e.g. Raspberry Pi) adjacent to the machine — enabling:

- Remote job submission from plat-trunk browser
- Real-time machine status (running, idle, error)
- Coil inventory monitoring
- Full design → manufacture → status loop

See [docs/adr-opcua-integration.md](docs/adr-opcua-integration.md) for the architecture.
Companion repo: https://github.com/joeblew999/opcua-howick (planned)

## Context

This crate was extracted from [plat-trunk](https://cad.ubuntusoftware.net) —
a browser-native B-Rep CAD platform built on the [Truck](https://github.com/ricosjp/truck)
Rust CAD kernel compiled to WASM.

The immediate use case: a steel framing factory in Thailand running Howick FRAMA machines,
currently using SketchUp + FrameBuilderMRD. plat-trunk aims to replace that pipeline with
precise B-Rep geometry and direct machine output.
